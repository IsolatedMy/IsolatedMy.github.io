在C++程序中调用MLIR函数

发表于 2022-10-23 更新于 2022-10-24 分类于 article

这篇文章介绍如何在C++程序中调用MLIR函数。

这篇文章采用的思想是将MLIR程序打包成库文件，将C程序和生成的库链接起来。链接方式取决于生成的库是动态链接库还是静态链接库。这种方法的优点在于它不仅适用于MLIR与C的链接，想要在C中调用其他方式实现的函数也可以利用这种方法处理。实际上，MLIR项目中可能有提供执行将输入的MLIR程序执行的C接口，这一点我没有深究，似乎可以参考这个文件：CRunnerUtils.h。

¶实现过程

首先需要了解什么是目标代码(object code)。目标代码是指计算机科学中编译器或者汇编器处理源代码后所生成的代码，它一般由机器代码或者接近于机器语言的代码组成。目标文件即存放目标代码的计算机文件，它常被称为二进制文件(binaries)。目标文件是从源代码文件产生程序文件这一过程的中间产物 - 维基百科。

所以，为了生成最后的可执行文件，应该先生成MLIR代码转换成目标代码，然后与C++程序链接。为了理解这一过程，我先调研了gcc和clang生成库的方法。

这篇博客中提到了gcc制作静态库的过程。大概的制作步骤为

  1. 生成对应的目标代码。例如：gcc sum.c -c sum.o
  2. 将生成的目标代码打包，这里利用ar工具，使用ar rcs + <静态库的名称> + <所有生成的.o文件>
    • GNU的ar指令用于建立、改变*打包文件(archive)*和从打包文件中抽取 - Linux man page；
    • -r是必要参数，用于指示将文件插入到打包文件当中；
    • c和s是选项参数，分别表示“建立打包文件”和“建立打包文件的符号表”

关于clang工具生成库的方法，参考这个问题 - Create STATIC and SHARED libraries with Clang。

  1. 首先独立编译源文件：clang -c -o lib_source.o lib_source.c -fPIC
  2. 如果对于Linux上的静态库，将所有.o文件打包在一起：ar r library.a lib_source.o
  3. 如果对于Linux上的共享库，利用-shared选项链接：clang -shared -o library.so lib_source.o

MLIR转换成目标代码的方式参考这个问题 - How to convert LLVM IR to object code?。步骤如下：

  1. 记待处理的程序为main.mlir。利用mlir-opt指令将程序降低为LLVM方言级别，例如mlir-opt --convert-std-to-llvm main.mlir -o main_llvm.mlir。添加的转换方言的编译选项根据具体处理程序发生变化；
  2. 将LLVM方言转换成LLVM字节码：mlir-translate --mlir-to-llvmir main_llvm.mlir -o main_llvm.bc；
  3. 使用LLVM静态编译器生成对象代码：llc main_llvm.bc -o main.llvm.o -filetype=obj

注意，这里使用的指令是旧版本的指令。另外，MLIR官方文档 - LLVM IR Target一节中详细描述了MLIR程序到LLVM IR的转换过程。

关于mlir程序和C++程序的链接方式，我本来想参照上述的目标代码生成方法一步步探索。然后，我看到问题 - How to compile and link with other c/c++ programs（下面简记为问题A）中讨论的内容基本就是我想做的。其中有两点值得注意：

  1. MLIR的memref结构转换成C++类型。之前提到的官方文档中提到，含秩内存引用类型(Ranked Memref Types)会被转换成5个域的描述符：

    • 指向分配的数据缓冲的指针，称作“分配指针(allocated pointer)”；
    • 指向memref索引的正确对齐数据指针的指针，称作“对齐指针(aligned pointer)“；
    • 一个降低且转换过的index类型整数，包含了memref访问的第一个元素距离缓冲区开始位置的距离，以元素数量表示，被称为“偏移(offset)”；
    • 一个包含与memref的维数(rank)一样多的转换后的index类型整数：数列用于表示大小，即沿着指定维度的元素数量；
    • 另一个包含与memref的维数一样多的转换后的index类型整数：第二个数列用于表示“步长”（在张量抽象意义上），即为获取下一个逻辑索引的元素需要越过的底层缓冲中连续元素的数量。

    描述符的例子可以看llvm-project/CRunnerUtils。在问题A中也提供了一个结构体表达描述符。

  2. 另一个是函数名。如果不对原本的mlir程序做额外处理，直接将生成的LLVM IR程序转换成目标代码，链接后的可执行程序就会产生段错误。为此，如问题A所说，需要通过C wrapper对函数包装，添加在之前的官方文档中提到的属性。这一点可以通过对mlir-opt命令增加llvm-request-c-wrppers完成。
